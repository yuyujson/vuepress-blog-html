语雀写作,自动部署gitpage

# 前言

先说一下我之前从写博客到发布的过程

  1. 语雀写 md 文档是真的爽, 我写博客一般先在语雀写, 图片也直接复制至语雀
  2. 写完后再从语雀导出, 然后将图片一个一个下载到本地项目中,然后在 md 文档中进行替换
  3. 上传至 github, 并部署 gitPage 过程十分麻烦, 特别是将图片保存到本地特别痛苦, 然后就想到是否可以将语雀文章通过直接同步至 hexo,经过一番周折终于搞出来了, 惬意~

# 涉及到的工具

  1. hexo
  2. yuque-hexo 项目 https://github.com/x-cold/yuque-hexo (opens new window)
  3. github
  4. gitpage
  5. travis-ci (持续集成服务)
  6. 语雀

# 步骤

  1. 第一版是按照 hexo 博客终极玩法 (opens new window) 来搭建的, 不过身为技术小白, 搞不太懂里面的功能,很多地方写的很模糊
  2. 上面那个先看看, 然后看戴爱玲的博客 (opens new window) , 这篇写的特别详细

嗯, 照着上面弄就能弄出来, 就不写了, 反正是前端的东西 QAQ 用了两天, 经常会报错, GitHub 地址无法解析, 放弃了, 我删除了语雀调用腾讯云函数的链接, 并把.travis.yml文件中的script:中参数修改为了hexo g, 不再自动同步语雀内容. 现在的流程是:

  1. 语雀写博客
  2. 在本地运行yuque-hexo sync 进行同步
  3. 提交代码至 github, travis-ci自动部署页面
